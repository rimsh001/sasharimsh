# Инструкция по развёртыванию на GitHub Pages

## Что нужно перед началом

- Аккаунт на GitHub (https://github.com)
- Установленный Git на компьютере (или GitHub Desktop)
- Доступ к настройкам домена `sasharimsh.ru` у регистратора (Reg.ru, NIC.ru, Cloudflare и т.д.)

---

## Шаг 1. Создать репозиторий

1. Зайти на https://github.com и нажать **New repository**.
2. Имя репозитория — `sasharimsh.ru` (или любое другое — это не влияет на работу домена).
3. Visibility — **Public** (для бесплатного GitHub Pages нужен публичный репозиторий).
4. **Не** добавлять README, .gitignore, лицензию (они уже есть в нашей папке).
5. Нажать **Create repository**.

---

## Шаг 2. Загрузить файлы

### Вариант А — через GitHub Desktop (проще)

1. Установить GitHub Desktop: https://desktop.github.com
2. Войти под своим аккаунтом.
3. **File → Clone repository** → выбрать только что созданный репозиторий → клонировать в удобную папку.
4. Скопировать **всё содержимое** папки сайта (включая скрытые файлы `.gitignore`, `CNAME`) в папку клона.
5. В GitHub Desktop появятся изменения. Внизу слева — поле для commit message. Написать «Initial commit».
6. Нажать **Commit to main**.
7. Нажать **Push origin** вверху.

### Вариант Б — через командную строку

```bash
cd путь/к/папке/сайта

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_USERNAME/sasharimsh.ru.git
git push -u origin main
```

---

## Шаг 3. Включить GitHub Pages

1. На странице репозитория нажать **Settings**.
2. В левом меню — **Pages**.
3. **Source** → выбрать **Deploy from a branch**.
4. **Branch** → выбрать **main** и папку **/ (root)**.
5. Нажать **Save**.
6. Через 1–2 минуты GitHub соберёт сайт и покажет URL вида `https://ВАШ_USERNAME.github.io/sasharimsh.ru/`.
7. Проверить, что сайт работает по этому URL.

---

## Шаг 4. Привязать домен sasharimsh.ru

### 4.1 На стороне GitHub

1. В **Settings → Pages** в поле **Custom domain** ввести `sasharimsh.ru`.
2. Нажать **Save**.
3. GitHub проверит DNS и покажет ошибку «DNS check failed» — это нормально, мы ещё не настроили DNS.
4. Файл `CNAME` уже лежит в репозитории, поэтому привязка сохранится.

### 4.2 На стороне регистратора домена

Зайти в личный кабинет регистратора (там, где куплен sasharimsh.ru). Найти раздел **DNS-записи** или **Управление DNS**.

**Удалить** все существующие A-записи для @ (корня домена) и CNAME для www.

**Добавить четыре A-записи** для корня @ (это IP-адреса GitHub Pages):

| Тип | Имя | Значение |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

**Добавить CNAME для www:**

| Тип | Имя | Значение |
|---|---|---|
| CNAME | www | ВАШ_USERNAME.github.io |

(вместо `ВАШ_USERNAME` — ваш логин на GitHub).

Сохранить изменения.

### 4.3 Подождать обновления DNS

DNS обновляется от 15 минут до 24 часов. Обычно — 1–4 часа. Можно проверить статус через сервисы вроде https://dnschecker.org/ — ввести `sasharimsh.ru` и тип A.

### 4.4 Включить HTTPS

Когда DNS-записи распространятся:
1. Вернуться в **Settings → Pages**.
2. Внизу поставить галочку **Enforce HTTPS**.
3. Если галочка серая — подождать ещё, GitHub выпускает SSL-сертификат автоматически.

## Шаг 5. Проверить работу сайта

Открыть в браузере:
- `https://sasharimsh.ru/` — главная
- `https://sasharimsh.ru/about.html`
- `https://sasharimsh.ru/services.html`
- `https://sasharimsh.ru/article-1.html`

Проверить:
- ✓ Сайт открывается через HTTPS.
- ✓ Все страницы доступны.
- ✓ Картинки и стили загружаются.
- ✓ Кнопки «Записаться» ведут в Telegram.
- ✓ Сайт корректно отображается на мобильном.

## Шаг 6. Зарегистрировать в поисковиках

### Яндекс.Вебмастер
1. https://webmaster.yandex.ru/
2. **Добавить сайт** → ввести `https://sasharimsh.ru/`.
3. Подтвердить владение через DNS (TXT-запись) или через мета-тег.
4. После подтверждения — **Индексирование → Файлы Sitemap** → добавить `https://sasharimsh.ru/sitemap.xml`.

### Google Search Console
1. https://search.google.com/search-console
2. **Add property** → ввести `https://sasharimsh.ru/`.
3. Подтвердить владение (DNS или HTML-файл).
4. После подтверждения — **Sitemaps** → добавить `sitemap.xml`.

## Как обновлять сайт

### Через GitHub Desktop
1. Открыть проект в GitHub Desktop.
2. Внести изменения в файлы (например, поправить текст или добавить картинку).
3. В GitHub Desktop появятся изменения.
4. Внизу — commit message: написать что изменили (например, «Поправил тексты на главной»).
5. **Commit to main** → **Push origin**.
6. Через 1–2 минуты изменения появятся на сайте.

### Через командную строку

```bash
cd путь/к/папке/сайта
# вносим изменения
git add .
git commit -m "Что изменили"
git push
```

## Возможные проблемы

### Сайт не открывается через несколько часов
- Проверить DNS-записи через https://dnschecker.org
- Проверить, что в Settings → Pages указан custom domain `sasharimsh.ru`
- Проверить, что в репозитории есть файл `CNAME` с одной строкой `sasharimsh.ru`

### Не работают стили или JS
- Убедиться, что в HTML пути относительные (`css/style.css`, не `/css/style.css` на главной)
- Проверить, что папки `css/` и `js/` загружены в репозиторий

### Ошибка 404 при переходе по ссылкам
- Убедиться, что все .html-файлы в корне репозитория, не в подпапках

### Картинки не отображаются
- Проверить, что папка `images/` загружена и содержит файлы
- Проверить, что имена файлов совпадают с указанными в HTML (с учётом регистра)

## Альтернатива: Cloudflare Pages

Если хочется более продвинутого хостинга с CDN, аналитикой и быстрой загрузкой:

1. https://pages.cloudflare.com
2. Connect to Git → выбрать репозиторий `sasharimsh.ru`.
3. Build settings: **None** (это статика).
4. Custom domain → ввести `sasharimsh.ru`.
5. Cloudflare автоматически настроит DNS и SSL.

Это даёт лучшую скорость, особенно для аудитории за пределами России.
